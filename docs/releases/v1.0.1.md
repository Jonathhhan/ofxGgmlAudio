# ofxGgmlAudio v1.0.1

## Compatibility

- openFrameworks: `of_v20260406_vs_64_release`
- minimum `ofxGgmlCore`: `1.0.1` / commit `2171e8b`
- `OFXGGML_AUDIO_VERSION_STRING`: `1.0.1`

## Highlights

- Added independent Audio addon version metadata.
- Exposed version metadata through `ofxGgmlAudio.h`.
- Kept Whisper as the first opt-in backend inside the broader Audio addon.
- Documented the release checklist and release policy for audio workflows.
- Preserved the root-level `ofxGgmlAudioTranscribeExample` as the first useful
  example lane.

## Runtime Scope

This release includes backend-neutral audio request/result types, stream
chunking, lightweight feature extraction, baseline VAD, WAV preparation helpers,
Whisper setup scripts, an unavailable-by-default Whisper backend boundary, and a
transcription example lane.

It does not claim general media decoding, denoising, voice conversion, emotion
detection, or audio event detection yet. Those remain roadmap tasks.
